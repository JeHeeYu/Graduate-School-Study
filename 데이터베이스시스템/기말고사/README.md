
## 데이터베이스 설계 절차 요약

|단계|핵심 키워드|산출물|
|:---:|:---:|:---:|
|1단계|요구사항 분석|요구사항 명세서|
|2단계|개념적 설계|ER 다이어그램|
|3단계|논리적 설계|릴레이션 스키마|
|4단계|물리적 설계|물리적 스키마|
|5단계|구현|SQL, DDL, DB 생성|

<br>

### 1단계 요구사항 분석
- 목적 : DB 용도 파악
- 방법 : 면담, 설문, 문서 분석
- 결과물 : 요구사항 명세서
- 포인트
  - 회원은 무엇으로 식별하는가? -> 회원 번호
  - 게시글은 누구와 관계 있는가? -> 회원

요약 : 요구사항 분석 단계는 사용자의 요구사항을 수집·분석하여 데이터베이스의 용도를 파악하고 요구사항 명세서를 작성하는 단계
 
<br>

### 2단계 개념적 설계
- 관점 : 업무 개념 관점, DBMS 독립적
- 목적 : 요구사항 분석 결과를 개념적인 데이터 구조로 표현
- 사용 모델 : E-R 모델
- 결과물
  - 개념적 스키마
  - E-R 다이어그램
- 핵심 구성 요소
  - 개체(Entity)
  - 속성(Attribute)
  - 관계(Releationship)
 
요약 : 개념적 설계 단계는 DBMS에 독립적으로 E-R 모델을 이용하여 개체, 속성, 관계를 정의하는 단계

<br>

### 3단계 논리적 설계
- 관점 : 데이터 모델 관점, DBMS 특성 고려
- 목적 : 개념적 스키마를 DBMS에 적합한 논리적 구조로 변환
- 주요 작업
  - ERD -> 릴레이션 스키마 변환
    - 개체(Entity) -> 테이블(Table)
    - 속성(Attribute) -> 컬럼(Column)
    - 식별자 -> 기본키(PK)
    - 관계 변환 규칙
      - 1:N 관계 -> N쪽에 외래키(FK)
      - N:M 관계 -> 관계용 테이블 생성
      - 1:1 관계 -> 한쪽에 외래키 추가
      - 다중값 속성 -> 별도 테이블
  - 기본키(PK), 외래키(FK) 결정
    - 각 테이블의 식별자(PK) 선정
    - 테이블 간 외래키(FK)로 표현
    - 참조 무결성 유지
  - 정규화 수행
    - 데이터 중복 최소화
    - 이상(Abnormaly) 방지
    - 정규화 단계
      - 제1정규형(1NF) : 모든 속성은 원자값
      - 제2정규형(2NF) : 부분 함수 종속 제거
      - 제3정규형(3NF) : 이행적 함수 종속 제거
- 결과물 : 논리적 스키마

요약 : 논리적 설계 단계는 개념적 스키마를 DBMS 특성을 고려하여 릴레이션 스키마로 변환하는 단계

<br>

### 4단계 물리적 설계
- 관점 : 실제 저장 구조 관점, DBMS, H/W, OS 고려
- 목적 : 논리적 스키마를 실제 저장 가능한 구조로 설계
- 주요 내용
  - 저장 구조
  - 인덱스
  - 접근 경로
  - 파일 구조
- 결과물 : 물리적 스키마

요약 : 물리적 설계 단계는 논리적 스키마를 기반으로 DBMS 및 하드웨어, 운영체제 특성을 고려하여 물리적 저장 구조를 설계하는 단계

<br>

### 5단계 구현
- 관점 : 실행/운영 관점, DBMS 사용
- 목적 : 설계된 내용을 실제 DB로 구현
- 주요 작업
  - SQL DDL 작성
  - 데이터베이스 생성
  - 테이블 생성
  - 초기 데이터 적재
- 결과물  
  - SQL DDL 코드
  - 실제 데이터베이스
 
요약 : 구현 단계는 SQL 문을 작성하여 데이터베이스와 테이블을 실제로 생성하고 데이터를 입력하는 단계

<br>
<br>

## 정규화와 이상현상
이상 현상(Abnormaly) : 불필요한 데이터 중복으로 인해 릴레이션에 대한 데이터 삽입/수정/삭제 연산을 사용할 때 발생할 수 있는 부작용
<br>
정규화(Normalization) : 이상 현상을 제거하면서 데이터베이스를 올바르게 설계해 나가는 과정

<br>

이상 현상 종류
- 삽입 이상
- 갱신 이상
- 삭제 이상
