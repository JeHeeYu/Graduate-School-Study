
# 1. 유용성 사용성 신뢰성 개념

## 유용성(Usefulness)

정의 : 유용성은 사용자의 목적 달성을 위해 도움을 제공하는 기능적 가치이며, 수단이 아닌 결과 중심으로 실제 과업 수행에서 병목을 해결하는 것
<br>
항목
- 사용자 목적 → 도움 → 기능적 가치
- 수단이 아닌 결과 중심
- 실제 과업 수행에서 병목 해결

<br>
<br>

## 사용성(Usability)

정의 : 사용성은 사용자가 시스템을 인지적으로 수월하게 사용할 수 있는 정도로, 수행 시간과 오류 등으로 정량적으로 측정되며 학습 없이도 사용할 수 있는 인터페이스를 지향
<br>
항목
- 편리함을 넘어선 인지적 수월함
- 정량적 측정 가능 (수행 시간, 오류 등)
- 학습 없이 사용 가능한 인터페이스

<br>
<br>

## 신뢰성(Trust/Reliability)
정의 : 신뢰성은 시스템이 정확하고 안정적으로 작동하며, 신속한 피드백과 오류 없는 수행을 통해 사용자에게 믿음을 형성하는 것
<br>
항목
- 신속한 응답 → 신뢰 형성
- 오류 방지 및 정확한 수행
- 예측 가능성과 안정성

<br>
<br>

### 연결 의미
- 유용성 → 사용성 → 신뢰성
- 유용성(왜) → 사용성(어떻게) → 신뢰성(믿음)
- 유용성 → 사용 이유
- 사용성 → 사용 가능
- 신뢰성 → 믿고 지속 사용

<br>
<br>

# 2. 멘탈 모델과 심상 모델

## 멘탈 모델(Mental Model)

정의 : 사용자가 과거 경험을 바탕으로 시스템의 작동 방식을 예측하는 주관적 인지 구조
<br>
항목
- 사용자 기준
- 주관적
- 예측

<br>
<br>

## 심상 모델(Conceptual Model)

정의 : 설계자가 의도한 시스템의 작동 방식에 대한 논리적 구조
<br>
항목
- 설계자 기준
- 시스템 구조
- 실제 동작 원리

<br>
<br>

# 3. HCI 상호작용 흐름 구조

## 센싱(Sensing)
사용자와 환경의 정보를 수집하는 단계
- 입력 (터치, 음성, 센서 등)
- 상황(Context) 파악

<br>
<br>

## 처리(Processing)
수집된 데이터를 해석하고 판단하는 단계
- AI / 시스템 판단
- 의미 해석

<br>
<br>

## 반응(Actuating)
사용자에게 결과를 전달하는 단계
- 화면 변화
- 소리, 진동 등 피드백

<br>
<br>

### HCI는 센싱, 처리, 반응의 3단계 순환 구조로 이루어지며, 사용자와 시스템 간 상호작용이 반복적으로 이루어짐

<br>
<br>

<br>
<br>

# 4. 공간

## 문제 공간(Problem Space)

정의 : 사용자의 문제와 목표를 정의하는 공간 (Why)
<br>
구성 요소
- 현재 상태(Initial State)
- 목표 상태(Goal State)
- 수단/행동(Operator)
- 제약 조건(Condition)

<br>

요약 : 문제가 무엇인지 정의하는 단계

<br>
<br>

## 솔루션 공간(Solution Space)

정의 : 문제를 해결하기 위한 다양한 대안들의 집합 (How)
<br>
구성 요소
- 여러 해결 방법 존재
- 대안 탐색 + 선택
- Trade-off 발생

<br>

요약 : 어떻게 해결할지 찾는 단계

<br>
<br>

# 5. HCI 3단계 구조

- 감지(Sensing) → 데이터 입력 (사용자/환경 정보 수집)
- 추론(Reasoning) → 데이터 해석 및 판단
- 실행(Actuating) → 결과 출력 및 시스템 동작

<br>
<br>

## 6. 유효성 & 환경 적합성


### 유효성 (Effectiveness)

정의 : 목적을 얼마나 정확하게 달성했는가 (결과의 정확성)
<br>
항목
- 목적 달성의 정밀도
- 성공률 / 정확도
- 결과 중심 평가
<br>
요약 : 결과가 맞냐

<br>
<br>

### 환경 적합성 (Environmental Fit)

정의 : 시스템이 사용 환경에 맞게 적절하게 작동하는 정도
<br>
항목
- 물리적 환경 (소음, 공간 등)
- 사회적 환경
- 맥락(Context) 고려
<br>
요약 : 상황에 맞냐

<br>
<br>
