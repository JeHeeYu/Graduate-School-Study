
# 1. 유용성 사용성 신뢰성 개념

## 유용성(Usefulness)

정의 : 시스템이 사용자의 목적을 달성하기 위해 필요한 기능을 제공하는 정도
<br>
항목
- 사용자 목적 달성에 필요한 기능 제공
- 기능적 가치 중심

<br>
<br>

## 사용성(Usability)

정의 : 사용자가 시스템을 쉽고 효율적으로 사용할 수 있는 정도
<br>
항목
- 사용의 용이성
- 수행 시간, 오류 등 정량적 평가 가능
- 학습 용이성

<br>
<br>

## 신뢰성(Trust/Reliability)
정의 : 시스템이 오류 없이 안정적으로 동작하여 사용자가 믿고 사용할 수 있는 정도
<br>
항목
- 정확한 수행
- 안정성 및 일관성
- 예측 가능성

<br>
<br>

### 연결 의미
- 유용성 → 사용성 → 신뢰성
- (기능 제공 → 사용 가능 → 믿고 사용)

<br>
<br>

# 2. 멘탈 모델과 심상 모델

## 멘탈 모델(Mental Model)

정의 : 사용자가 과거 경험을 바탕으로 시스템의 작동 방식을 예측하는 주관적 인지 구조
<br>
항목
- 사용자 기준
- 주관적
- 예측

<br>
<br>

## 심상 모델(Conceptual Model)

정의 : 설계자가 의도한 시스템의 구조와 동작 방식에 대한 논리적 모델
<br>
항목
- 설계자 기준
- 시스템 구조
- 실제 동작 원리

<br>
<br>

# 3. HCI 상호작용 흐름 구조

## 센싱(Sensing)
사용자와 환경의 정보를 수집하는 단계
- 입력 (터치, 음성, 센서 등)
- 상황(Context) 파악

<br>
<br>

## 처리(Processing)
수집된 데이터를 해석하고 판단하는 단계
- AI / 시스템 판단
- 의미 해석

<br>
<br>

## 반응(Actuating)
사용자에게 결과를 전달하는 단계
- 화면 변화
- 소리, 진동 등 피드백

<br>
<br>

### HCI는 센싱 → 처리 → 반응의 순환 구조로 이루어짐

<br>
<br>

<br>

# 4. 공간

## 문제 공간(Problem Space)

정의 : 해결해야 할 문제와 목표를 정의하는 공간
<br>
구성 요소
- 현재 상태(Initial State)
- 목표 상태(Goal State)
- 수단/행동(Operator)
- 제약 조건(Condition)

<br>

요약 : 무엇을 해결할 것인가

<br>
<br>

## 솔루션 공간(Solution Space)

정의 : 문제 해결을 위한 다양한 대안들의 집합
<br>
구성 요소
- 여러 해결 방법 존재
- 대안 탐색 + 선택
- Trade-off 발생

<br>

요약 : 어떻게 해결할 것인가

<br>
<br>

# 5. HCI 3단계 구조

- 감지(Sensing) → 데이터 입력 (사용자/환경 정보 수집)
- 추론(Reasoning) → 데이터 해석 및 판단
- 실행(Actuating) → 결과 출력 및 시스템 동작

<br>
<br>

## 6. 유효성 & 환경 적합성


### 유효성 (Effectiveness)

정의 : 목적을 얼마나 정확하게 달성했는가 (결과의 정확성)
<br>
항목
- 목적 달성의 정밀도
- 성공률 / 정확도
- 결과 중심 평가
<br>
요약 : 결과가 맞냐

<br>
<br>

### 환경 적합성 (Environmental Fit)

정의 : 시스템이 사용 환경에 맞게 적절하게 작동하는 정도
<br>
항목
- 물리적 환경 (소음, 공간 등)
- 사회적 환경
- 맥락(Context) 고려
<br>
요약 : 상황에 맞냐

<br>
<br>
